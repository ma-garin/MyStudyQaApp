# 権利安全監査報告

## 目的と範囲

この文書は、QA Learning Trainerを個人制作の非公式教材として公開するための技術監査記録です。法的評価を確定するものではなく、公開前には問題・解説・レッスン本文の独立した人間レビューが必要です。

確認対象は2026年6月12日時点の未コミット作業ツリーです。公式PDFや外部教材の本文は取得・閲覧していません。

## 現行ファイルの確認結果

- アプリ名、HTML title、PWA表示名を `QA Learning Trainer` へ統一した。
- 現行ワークツリーにPDF、公式ロゴ、旧公式風画像の実ファイルはない。
- トップページ、設定画面、全ページ共通の免責表示に、個人制作・非公式・非認定である旨を表示している。
- JSTQB/ISTQBの名称は、免責、権利帰属、外部リンク、学習範囲の参照元メタデータに限定した。
- 問題文・選択肢・解説に含まれていた受験要件の設問を、一般的なテスト分析の独自問題へ差し替えた。
- 権利者名を含む旧アプリ名、旧サンプル試験名、旧学習目標フィールド、旧画像への実参照は残っていない。
- PDF拡張子と旧画像名の検索ヒットは、READMEやテスト内の禁止・検査文字列のみである。

## オフライン対応とプライバシー（追加実装）

- ルートに `sw.js` を追加し、PWA のオフライン対応を実装した。アプリシェルと同一オリジン資産をキャッシュし、ナビゲーションは network-first、静的資産は cache-first とする。
- GitHub Pages のサブパス配信に合わせ、登録（`js/registerSW.js`）とキャッシュ URL を相対パスで統一した。絶対 `/` は使用しない。
- Google Analytics（gtag）は Service Worker でキャッシュ・オフライン偽装せず素通しする。`questionsData_*` と `lessonsData_*` の大型データはプリキャッシュせずランタイムキャッシュとする。
- `VERSION` 定数とデプロイ時の旧キャッシュ削除で、削除済みファイルが配信され続ける事故を防ぐ。
- gtag を全 HTML ページで統一した（従来 `ai-quiz.html` のみ欠落していた）。
- アクセス解析・Cookie・取得しない情報・オプトアウト方法を説明する `privacy.html` を追加し、設定画面と全ページ共通フッターから到達できるようにした。

## 履歴対応の判断材料

現行ファイルから削除しても、過去のGitオブジェクトにはPDFや画像が残る可能性があります。履歴調査で再配布できないファイルが見つかった場合は、管理者承認のうえで `git filter-repo` による削除、または安全な現行ファイルだけを使う新規リポジトリ作成を検討します。

履歴改変はコミットID、既存clone、forkへ影響します。バックアップ、対象一覧、共同作業者への告知なしにforce pushを実行しません。外部キャッシュや既存cloneから自動的に削除される保証もありません。
