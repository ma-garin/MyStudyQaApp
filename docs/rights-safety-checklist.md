# 権利安全チェックリスト

GitHub Pagesへの公開、PWAの配布、リリース作成の前に、現行HEADと実際の配信成果物の両方を確認します。

## 必須確認

- [x] PDFが含まれていない（自動検証・Git履歴調査で確認）
- [x] 公式ロゴが含まれていない（自動検証で確認）
- [x] 公式シラバス本文が含まれていない（自動検証で確認）
- [x] 公式問題が含まれていない（全問題が `officialQuestionUsed: false`）
- [x] アプリ名が公式誤認を招かない（"QA Learning Trainer" に統一）
- [x] 権利者名を含む旧アプリ名が残っていない（自動検証で確認）
- [x] 非公式表示がある（README・トップページ・設定画面・全ページ共通フッター）
- [x] 公式リンクは外部リンクのみ（istqb.org、jstqb.jp へのリンクのみ）
- [x] 問題・解説が自作である（全問題 `sourcePolicy: "original"`、`reviewerStatus: "reviewed"`）
- [ ] Google検索されても公式教材に見えない（Pages実機確認時に確認）
- [x] GitHub Pages公開前に全ファイル検索済み（contentSafety.test.js で自動検証）

## コンテンツ確認

- [x] 公式の学習目標文、説明文、表、図を転載していない（制作者確認）
- [x] 公式文の翻訳、要約コピー、語句を置換した近似コピーがない（制作者確認）
- [x] 公式サンプル問題の題材、数値、固有名詞、設問構造、選択肢構成を流用していない（制作者確認）
- [x] 章番号、節番号、LOコード、Kレベル等は参照メタデータに限定している
- [x] 各問題に独自作成を示すメタデータとレビュー状態がある（全297問で確認）
- [x] 教材と解説が公式資料なしでも理解できる独自の実務説明になっている（制作者確認）

## 表示と配信確認

- [x] README、トップページ、設定画面、フッター相当箇所に免責文がある
- [x] HTML title、manifest、PWA表示名、説明文、OGPに「公式」「認定」「準拠」等の誤認表現がない
- [x] favicon、PWAアイコン、スクリーンショットが独自デザインである（qa-learning-icon.svg）
- [x] service workerやビルド成果物が削除済みファイルをキャッシュしない（`sw.js` はアプリシェルのみプリキャッシュし、`activate` で旧 `qa-app-*` を削除。`tests/serviceWorker.test.js` が実在を検証）
- [x] 外部リンクには公式情報を別サイトで確認する旨が明記されている（syllabus.html）
- [x] `npm test`または既存テストが成功している（12ファイル・80件成功）
- [x] HTML、JavaScript、CSS、manifest、service workerの参照切れがない（自動検証で確認）
- [x] 資格定義の問題・レッスンデータが実際に読み込め、件数とIDが整合している（contentSafety.test.js で確認）

## Git履歴確認

- [x] 現行HEADだけでなく過去のGit履歴にPDFや公式画像がないか調査した（コミット1件のみ、PDF・画像なし）
- [x] 過去に含まれていた場合の対応判断（該当なし・履歴書き換え不要）
- [x] 履歴改変が必要な場合の手続き（該当なし）
- [x] force pushを無断で実行していない
- [x] fork、clone、リリース、Pages成果物、キャッシュも個別に確認した（新規リポジトリのため該当なし）
